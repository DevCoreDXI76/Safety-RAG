# 안전보건교육일지 문서종류 생성 기능 — 설계 스펙

## 배경 및 목적
`docs/handoff_phase6_문서종류확장.md`에서 표준 작업계획서와 함께 다루려다
범위를 좁히며 "다음 단계"로 미뤄둔 항목이다. `/document-types` 메뉴에는
이미 "안전보건교육일지"가 존재하지만 생성 로직이 미완성이다. 표준
작업계획서의 `work_type` 직접주입 패턴(마커 기반 KB 섹션 추출)이 이미
검증되어 있으므로, 동일한 패턴을 재사용해 `education_type`/
`special_work_type` 파라미터를 추가한다.

지식베이스(`knowledge_base/안전보건교육_법정시간및내용.txt`)는 이미
저장소에 존재하며, 근로자/관리감독자 교육시간표·특별교육 5개 항목별
교육내용·건설업 기초안전보건교육이 정리되어 있다.

## 왜 RAG 검색이 아니라 직접주입인가
표준 작업계획서 개발 과정에서, 이 KB 파일을 RAG 유사도 검색에만 맡겼을 때
맨홀작업(§34호)과 밀폐공간작업(§35호)이 매 생성마다 다르게 나오거나
혼동되는 문제가 실제로 발생했다(top-k 순위가 project_info 문구에 따라
매번 달라짐). 같은 파일을 다루는 이번 기능에서 같은 실수를 반복하지
않기 위해, 처음부터 마커 기반 직접주입으로 설계한다.

## 확정된 결정사항 (브레인스토밍 중 확인)
1. **건설업 기초안전·보건교육(4시간) UI 처리**: 별도 토글 없이,
   "신규채용시교육" 선택 시 문서 본문에 항상 자동 포함한다.
2. **위험성평가 연동**: TBM·표준작업계획서와 동일하게, 같은 현장의 최근
   위험성평가 회차를 연동(`risk_assessment_id`)할 수 있게 한다.
3. **스코프 제외**: 정기교육·작업내용변경시교육은 핸드오프 문서 권고에
   따라 이번 라운드에서 구현하지 않는다.

## 변경 파일

### `common.py`
- `EDUCATION_TYPE_SECTION_MARKERS`: `"신규채용시교육"`/`"특별교육"` → 마커 쌍 리스트
- `SPECIAL_WORK_TYPE_EDUCATION_MARKERS`: 작업유형 5종 → 마커 쌍
- `get_education_type_context(education_type, special_work_type=None)`:
  `get_work_type_context()`와 동일 구조. `read_kb_file()`/`extract_kb_section()`
  재사용해 매칭 섹션들을 이어붙여 반환

### `generate_draft.py`
- `_build_generation_prompt()`/`generate_document_draft()`/
  `generate_document_draft_stream()`에 `education_type=None,
  special_work_type=None` 파라미터 추가 (work_type과 나란히)
- `is_education_with_type = document_type == "안전보건교육일지" and
  education_type in EDUCATION_TYPE_SECTION_MARKERS` 판정 추가
- 매칭 시 `exclude_sources`에 `EDUCATION_KB_FILE` 추가(work_type의
  `WORK_TYPE_KB_FILE` exclude와 동일 이유 — 직접주입과 RAG 중복 방지)
- 안정 구간(A-2 캐싱 대상)에 education 컨텍스트 포함 — work_type_text와
  동일하게 project_info에 좌우되지 않으므로 캐시 가능
- system_prompt에 조건부 지시 추가: 특별교육 75V 기준을 작업계획서
  50V/250VA 기준과 혼동하지 말 것, 맨홀(§34호)·밀폐공간(§35호)은 법령상
  별도 항목이므로 실제 작업 성격에 맞는 것만 정확히 인용할 것
- `education_type_line`(현장명/작업유형과 같은 패턴) user_prompt에 추가

### `api/schemas.py`, `api/routes.py`
- `GenerateRequest`에 `education_type: Optional[str] = None`,
  `special_work_type: Optional[str] = None` 추가
- `/generate`에서 `generate_document_draft_stream(...)` 호출 시 그대로 전달

### `webapp/index.html`
- "안전보건교육일지" 선택 시 교육종류 선택 섹션 노출(신규채용시교육/특별교육,
  `type-card` 패턴 재사용)
- "특별교육" 선택 시 작업유형 5종 하위 선택 섹션 추가 노출(2단계)
- `risk-section` 노출 조건에 안전보건교육일지 추가
- payload에 `education_type`, `special_work_type` 추가
- 생성 전 검증: 교육종류 미선택 시 에러, 특별교육인데 작업유형
  미선택 시 에러

### 신규 `test_education_citations.py`
`test_worktype_citations.py`와 동일한 컨벤션(plain script, API 비용
발생하는 온디맨드 스크립트). 케이스: 신규채용시교육, 특별교육×5작업유형.
기대 사실(각 교육시간·교육내용 키워드) 포함 여부와
`find_unverified_citations()` 경고 여부를 확인.

## 테스트 우선순위
신규채용시교육 → 특별교육(맨홀작업) → 특별교육(굴착작업) → 나머지
(전기작업/밀폐공간작업/철탑 등 조립·해체·변경작업)

## 검증 방법
1. `get_education_type_context()` 단위 호출로 마커 추출이 정확한지 직접 확인
   (덧붙는 내용 없이 딱 필요한 섹션만 나오는지 — work_type 때 있었던
   "프리앰블 교차오염" 버그 재발하지 않는지 점검)
2. `test_education_citations.py`로 6개 케이스(신규채용시교육 + 특별교육 5종)
   생성해 기대 사실 포함·인용 경고 없음 확인
3. 신규채용시교육 결과에 건설업 기초안전·보건교육(4시간) 안내가 항상
   포함되는지 확인
4. 맨홀작업/밀폐공간작업 케이스에서 서로 혼동되지 않고 각자 정확한
   조번호(§34호/§35호)로 구분되는지 확인
