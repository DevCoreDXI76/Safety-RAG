# 이격거리/간격 수치 가드레일 설계

- 작성일: 2026-07-13
- 관련 프로젝트: 안전서류 AI 초안 생성 시스템 (safety-rag)
- 배경: [docs/handoff_성능개선_및_검토사항.md](../../handoff_성능개선_및_검토사항.md) C-2
  ("활선설비 이격거리 30cm 순수 창작") 항목의 후속 조치. C-3(차량계
  하역운반기계 work_type 승격) 완료 후 재관찰한 결과, 원래 사례(활선설비
  이격거리)는 개선됐지만 동일 패턴(구체적 이격거리 수치 창작)이 다른
  위험요인(설비 간격)으로 옮겨간 것이 확인됨. "법조문 형식이 아닌 일반
  수치"에 대한 가드레일이 없다는 게 근본 원인으로 재확인됨.

## 배경 및 목적

이 프로젝트는 이미 두 종류의 가드레일을 갖추고 있다:

1. **조문/별표 인용 검증** (`find_unverified_citations`, `common.py`) — 초안에
   등장하는 "제○조/항/호", "별표○" 인용 중 참고자료에 없는 것을 탐지
2. **위험성 점수 구간표 표기 검증** (`find_broken_risk_score_ranges`,
   `common.py`) — KB에 정확히 명시된 "1~4"/"5~9"/"10~25" 표기가 물결표
   누락 등으로 깨졌는지 탐지

두 체커 모두 "법령 조문 번호"나 "고정된 판정 구간값"처럼 정답 집합이
명확한 영역만 다룬다. 반면 이격거리·간격 같은 **일반 수치**는 정답 집합이
고정돼 있지 않고 상황마다 달라서, 지금까지 어떤 가드레일도 감시하지
않았다. 이번 설계는 이 사각지대를 좁히는 세 번째 체커를 추가한다.

## 스코프

**감시 대상은 이격거리·간격 계열로 한정한다.** 인원수·무게·전압·온도 등
다른 종류의 일반 수치는 이번 스코프에서 제외한다 — 지금까지 실제로
관찰된 순수 창작 사례가 전부 "OOcm/OOm 이격/간격 확보" 패턴이었고,
스코프를 넓히면 오탐 위험이 커지기 때문이다(예: "포크 높이를 15~20cm로
제한"처럼 이격거리와 무관한 기계적 수치까지 걸릴 수 있음).

## 설계

### 탐지 알고리즘

기존 `find_unverified_citations()`와 동일한 방향으로 설계한다: **초안에
있는데 참고자료(reference_context)에 없으면 미검증으로 반환**. 위험성
점수 구간표 체커(`find_broken_risk_score_ranges`)와는 반대 방향인데,
그 체커는 KB에 고정된 정답 3개(1~4/5~9/10~25)가 있어서 "정답이 있는지"를
확인하는 방식이 성립했지만, 이격거리는 정답이 상황마다 달라서 고정
목록을 만들 수 없다. 대신 인용 검증과 동일하게 "출처 없는 값"을 잡는
방향이 맞다.

**앵커 키워드** (이 중 하나라도 포함된 텍스트 조각에서만 수치를 추출):
`이격거리`, `이격`, `간격`, `여유 공간`, `안전거리`, `접근 한계거리`

`접근 한계거리`를 포함시킨 이유: 전기작업 KB
(`표준작업계획서_법정별표.txt`)에 이미 등장하는 표현인데 구체적인 수치는
없다. "접근 한계거리(50cm 이상)"처럼 이 문구에 숫자를 갖다 붙이는 패턴을
정확히 잡기 위한 앵커다.

**수치+단위 정규식**:
```python
DISTANCE_VALUE_PATTERN = re.compile(
    r"\d+(?:\.\d+)?\s*(?:cm|mm|m|미터|센티미터|센티)(?![a-zA-Z가-힣])"
)
```
- 단위 알터네이션 순서(`cm|mm|m|...`)가 중요하다 — `m`을 먼저 두면 `mm`,
  `cm`의 일부만 매칭되는 문제가 생긴다.
- 부정 전방탐색(`(?![a-zA-Z가-힣])`)으로 다른 단어에 파묻힌 부분 매칭을
  방지한다.

**청크 분리 방식**: draft를 문장이 아니라 `<br\s*/?>|\n` 기준으로
조각낸다. 실제 생성 결과물은 "① [공학적 대책] ... <br>② [관리적 대책]
..." 처럼 표 셀 안에서 `<br>` 태그로 항목을 이어붙이는 형식이라, 문장
단위(마침표 기준)로 자르면 소수점 있는 수치("1.5m")가 깨지거나 항목
경계를 놓칠 수 있다. `<br>`/줄바꿈 단위가 이 프로젝트의 실제 출력
형식에 더 안전하다.

**함수 시그니처** (`common.py`, `find_unverified_citations` 바로 아래에
추가):
```python
CLEARANCE_KEYWORDS = ["이격거리", "이격", "간격", "여유 공간", "안전거리", "접근 한계거리"]

def find_unverified_clearance_values(draft, reference_context):
    """
    draft에서 이격거리/간격 관련 키워드 근처에 등장하는 구체적인 수치
    (예: "30cm")를 추출해, reference_context에 그대로 등장하지 않는
    것만 반환. find_unverified_citations와 동일하게 소프트 경고 용도.
    """
    chunks = re.split(r"<br\s*/?>|\n", draft)
    draft_values = set()
    for chunk in chunks:
        if any(kw in chunk for kw in CLEARANCE_KEYWORDS):
            draft_values.update(
                _normalize_citation(m.group())
                for m in DISTANCE_VALUE_PATTERN.finditer(chunk)
            )
    if not draft_values:
        return []
    context_values = {
        _normalize_citation(m.group())
        for m in DISTANCE_VALUE_PATTERN.finditer(reference_context)
    }
    return sorted(draft_values - context_values)
```
- 정규화는 기존 `_normalize_citation`(공백 제거)을 그대로 재사용한다 —
  이격거리 값에는 조/항/호 같은 구성요소가 없으므로 그대로 재사용해도
  의미가 달라지지 않는다.
- context에 앵커 키워드가 없어도(=이 KB가 이 사안에 대해 아예 언급이
  없어도) draft에 값이 있으면 그대로 "미검증"으로 잡힌다 — 이게 의도한
  동작이다(문맥 자체가 없다는 것 자체가 창작의 강한 신호).

### 통합 지점

`generate_draft.py`의 `_finalize_draft`에 기존 두 체크와 나란히 세 번째
체크로 추가한다. 기존 두 체커와 동일한 패턴(경고 문구를 draft에 append,
`warning` 변수에 누적)을 따른다.

### system_prompt 가드레일 추가

기존 (A)/(B) 조문 규칙 문단(참고자료에 있는 조문은 반드시 인용, 없는
조문은 지어내지 말 것) 바로 뒤에, 이격거리·간격 수치에도 동일한 원칙을
적용하라는 지시를 추가한다:

> "이격거리·간격 등 안전거리 수치를 다룰 때도 위 (A)/(B) 규칙을 그대로
> 적용해. 참고자료에 구체적인 거리·간격 값이 명시되어 있으면 그 값을
> 그대로 사용하고, 명시되어 있지 않으면 구체적인 숫자를 만들어내지 말고
> '현장 실측 후 확보'처럼 서술해."

## 오탐 방지 근거 (사전 검증)

기존 재관찰 테스트에서 실제로 확보한 초안 텍스트(활선설비는 안전하게
서술됨, 위험요인 6의 "30cm"는 창작)를 이용해 API 재호출 없이 로컬로
검증한다. 기대 결과:
- 위험요인 6 "통로 폭과의 여유 공간(좌우 각 최소 30cm 이상)" → "30cm"
  탐지됨(앵커 "여유 공간" 포함, KB에 근거 없음)
- 위험요인 3 "이격거리를 사전 실측하고..." → 수치 자체가 없으므로 탐지
  대상 없음
- 위험요인 1 "포크 높이를... 15~20cm로 제한" → 앵커 키워드 없음, 애초에
  검사 대상이 아님(스코프 밖)
- 굴착작업 KB의 "매설물 인근... 수평 1m 이내" → 앵커 "이격거리"가 KB
  원문 표제에 있고, 컨텍스트에도 동일하게 "1m"이 존재하므로 미검증
  목록에 안 잡힘(정상 통과)

## 테스트 계획

- **로컬(무비용) 검증**: 위 4가지 케이스를 하드코딩한 샘플 텍스트로
  함수 단위 확인 (API 호출 없음)
- **API 회귀 확인**: 별도로 요청하지 않는 한 이번 스펙 구현 직후에는
  실행하지 않는다 — 기존 관례대로 API 비용이 드는 검증은 사전에 제안
  후 사용자 승인을 받는다

## 알려진 한계 (오탐/누락 가능성)

- 표기 변형에 취약하다 — KB가 "1m"로 쓰는데 초안이 "1미터"로 바꿔
  쓰면 의미상 같아도 문자열이 달라 오탐(미검증)으로 잡힐 수 있다. 기존
  인용 체커도 동일한 성격의 트레이드오프를 이미 받아들이고 있다(소프트
  경고라 과잉 경고가 누락보다 안전하다는 원칙).
- 앵커 키워드 밖의 이격거리 창작(예: 키워드 없이 그냥 "30cm 확보"라고만
  쓰는 경우)은 놓칠 수 있다. 이번 스코프는 실제로 관찰된 패턴(키워드 +
  수치 조합)만 우선 방어한다.
