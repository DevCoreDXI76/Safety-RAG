# 베타 폼 버그 수정 (복사 실패 대응 + 편집 기능) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**권장 실행 순서**: 이 플랜은 [2026-07-22-xlsx-내보내기.md] 다음에 실행한다. 이유: (1) `webapp/index.html`의 `result-meta` 버튼 영역과 `streamResult()`의 `"done"` 분기를 그 플랜이 먼저 수정하며, 이 플랜은 그 수정된 코드를 기준으로 이어서 수정한다. (2) 이 플랜의 수동 검증도 그 플랜에서 만든 `dev_login_helper.py`를 그대로 사용한다.

**Goal:** 베타0 피드백(`docs/strategy/Safety-RAG_베타0_피드백_분석_및_게이트판단_2026-07-22.md`)에서 보고된 "자동전송 실패(4회 재시도)"와 "편집 어려움" 두 가지를 해결한다.

**조사 결과로 확정한 범위**: `webapp/index.html`에는 Telegram `sendData()`/공유 API가 전혀 없고, 결과를 밖으로 내보내는 유일한 수단은 클립보드 복사 버튼(`btn-copy`, 기존 `webapp/index.html:721-725`)뿐이었다. 사용자가 말한 "전송 실패"는 실제로는 이 복사 버튼이 텔레그램 인앱 WebView에서 실패했을 가능성이 매우 높다(Clipboard API는 WebView 환경·권한에 따라 조용히 예외를 던질 수 있음). "파일 다운로드 불가"는 별도 기능 부재였으므로 [2026-07-22-xlsx-내보내기.md]에서 해결했고, 이 플랜에서 다시 다루지 않는다. "편집 어려움"은 결과 영역(`result-body`)이 읽기 전용 렌더링 HTML이라 애초에 편집 UI가 없었기 때문이므로, 편집 가능한 텍스트 뷰를 추가한다.

**Architecture:** (1) 복사 버튼에 Clipboard API 실패 시 `document.execCommand("copy")` 폴백을 추가하고, 성공/실패를 사용자에게 명확히 표시한다. (2) 결과 영역에 "편집" 토글을 추가해 렌더링된 뷰와 원본 Markdown을 담은 `<textarea>`를 전환할 수 있게 한다. 편집한 내용은 복사 버튼과 미리보기 렌더링에 반영되지만, 서버에 저장된 레코드(및 XLSX 다운로드가 참조하는 레코드)는 수정하지 않는다 — 이건 의도적인 스코프 제한이다 (재생성 없는 서버측 레코드 수정은 별도 백�로그 항목).

**Tech Stack:** 순수 프론트엔드 JS/CSS (`webapp/index.html`), 신규 백엔드 변경 없음.

## Global Constraints

- 이 저장소에 자동화된 프론트엔드 테스트 프레임워크(Playwright 등)가 설정돼 있지 않으므로, 검증은 [2026-07-22-xlsx-내보내기.md] Task 4에서 만든 `dev_login_helper.py`를 이용한 수동 브라우저 확인으로 한다.
- 편집 기능은 클라이언트 상태(`state.lastRawMarkdown`)만 갱신한다 — 서버 레코드 재저장 API를 새로 만들지 않는다(YAGNI, 베타0 게이트 판단에서 무거운 인프라 작업은 보류하기로 했음).

`showError()` 함수 아래에 새 함수 추가:

```javascript
  // 텔레그램 인앱 WebView 등에서는 navigator.clipboard.writeText()가 권한/보안
  // 컨텍스트 문제로 조용히 실패할 수 있다(베타0 "자동전송 실패" 피드백의 원인으로 추정).
  // execCommand("copy") 폴백을 두고, 성공/실패를 버튼 텍스트로 명확히 알린다.
  async function copyResultText() {
    const text = (state.lastRawMarkdown != null) ? state.lastRawMarkdown : $("result-body").innerText;
    let succeeded = false;

    try {
      if (navigator.clipboard && navigator.clipboard.writeText) {
        await navigator.clipboard.writeText(text);
        succeeded = true;
      }
    } catch (e) {
      succeeded = false;
    }

    if (!succeeded) {
      const fallback = document.createElement("textarea");
      fallback.value = text;
      fallback.style.position = "fixed";
      fallback.style.opacity = "0";
      document.body.appendChild(fallback);
      fallback.focus();
      fallback.select();
      try {
        succeeded = document.execCommand("copy");
      } catch (e2) {
        succeeded = false;
      }
      document.body.removeChild(fallback);
    }

    $("btn-copy").textContent = succeeded ? "복사됨" : "복사 실패 (직접 선택해 복사하세요)";
    setTimeout(() => { $("btn-copy").textContent = "복사"; }, 1500);
  }
```
