# 문서 생성 순서 안내 인앱 UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**권장 실행 순서**: [2026-07-22-xlsx-내보내기.md] 다음, [2026-07-22-베타폼-버그수정.md]와는 독립적으로 실행 가능(둘 다 `webapp/index.html`을 건드리지만 다른 섹션이라 충돌 없음). 단, 수동 검증에는 [2026-07-22-xlsx-내보내기.md] Task 4가 만든 `dev_login_helper.py`가 필요하다.

**Goal:** 콘텐츠는 이미 기획된 상태(`docs/strategy/Safety-RAG_문서생성순서가이드_콘텐츠기획.md`)지만 화면에 반영되지 않아 베타0 응답자 2/2명이 문서 생성 순서(위험성평가표 → 표준 작업계획서 → TBM일지)를 놓친 문제를 해결한다.

**조사 결과로 확정한 범위**: `api/routes.py`/`generate_draft.py`에는 위험성평가 기록을 다음 문서 생성에 연동하는 배관(`risk_assessment_id`, `get_record_by_id()`, `linked_risk_context`)이 이미 있고, 프론트엔드의 `loadRiskAssessments()`/`renderRiskList()`(`webapp/index.html:551-596`)는 기본값으로 **가장 최근 회차를 자동 선택**한다. 즉 "방금 만든 위험성평가표를 다음 문서에 자동으로 이어붙이는" 백엔드 로직은 이미 작동 중이며, 빠진 것은 순수하게 프론트엔드 안내 UI뿐이다. 콘텐츠 기획 문서가 제안한 항목 중 "공사 종류 변경 감지 → 위험성평가표 갱신 유도"는 문서 자체가 "아직 실제 베타 인터뷰로 검증되지 않은 가설"이라고 명시하고 있고, 텍스트 휴리스틱 감지가 필요한 별도 규모의 작업이라 이번 스코프에서 의도적으로 제외한다(백로그로 남김).

**Architecture:** (1) "문서 종류" 섹션 상단에 순서 안내 배너 추가(1회만 보이도록 `localStorage`로 닫기 상태 기억). (2) 위험성평가표 생성 완료 직후 결과 영역에 "표준 작업계획서 이어서 만들기" CTA 버튼 추가 — 클릭 시 기존 `selectDocumentType()`을 재사용해 표준 작업계획서를 선택하고, 기존 `loadRiskAssessments()`의 "최신 회차 자동 선택" 로직이 방금 만든 기록을 그대로 골라주도록 한다(새 로직 불필요).

**Tech Stack:** 순수 프론트엔드 JS/CSS (`webapp/index.html`), 신규 백엔드 변경 없음.

## Global Constraints

- 이 저장소에 자동화된 프론트엔드 테스트 프레임워크가 없으므로, 검증은 [2026-07-22-xlsx-내보내기.md] Task 4에서 만든 `dev_login_helper.py`를 이용한 수동 브라우저 확인으로 한다.
- "공사 종류 변경 감지" 휴리스틱은 이번 스코프에 포함하지 않는다(위 배경 설명 참고, YAGNI — 미검증 가설).

- [ ] **Step 1: CTA 렌더 함수 추가**

`toggleEdit()` 함수 아래(또는 [2026-07-22-베타폼-버그수정.md]를 실행하지 않았다면 `showError()` 함수 아래)에 추가:

```javascript
  // 위험성평가표 생성 직후에만 노출: 표준 작업계획서를 선택하면
  // loadRiskAssessments()의 기존 "최신 회차 자동 선택" 로직이 방금 만든
  // 위험성평가표를 그대로 골라주므로, 여기서는 문서종류만 바꿔주면 된다.
  function offerNextDocumentCta() {
    const existing = document.getElementById("order-guide-cta");
    if (existing) existing.remove();

    const workPlanType = state.documentTypes.find((t) => t.label === "표준 작업계획서");
    if (!workPlanType) return;

    const cta = document.createElement("div");
    cta.className = "order-guide-cta";
    cta.id = "order-guide-cta";
    cta.innerHTML = `
      <p>이 위험성평가표 내용을 이어서, <strong>표준 작업계획서</strong>도 바로 만들어볼까요?</p>
      <button class="btn-secondary" id="btn-next-doc-cta">표준 작업계획서 이어서 만들기</button>
    `;
    $("result-section").appendChild(cta);

    document.getElementById("btn-next-doc-cta").addEventListener("click", () => {
      cta.remove();
      selectDocumentType(workPlanType.id, workPlanType.label);
      $("worktype-section").scrollIntoView({ behavior: "smooth", block: "start" });
    });
  }
```
