## 2. 일반 사용자 흐름

### 2-1. 최초 접속 (승인 전)

1. 텔레그램에서 `@safety_rag_bot` 대화창을 열고, 메뉴 버튼("서류 생성")을 클릭해 미니앱을 엽니다.
2. 미니앱 화면이 뜨자마자 자동으로 백엔드에 "문서 종류 목록" 요청이 나가고, 이 시점에 시스템이 아직 승인되지 않은 사용자임을 확인합니다.
3. **이 요청 한 번으로 자동 처리됩니다** — 별도로 `/start`를 직접 입력할 필요 없음:
   - 대기 목록에 자동 등록
   - 신청자 본인에게 "사용 신청이 접수되었습니다. 관리자 승인 후 이용 가능합니다." 메시지 발송
   - 관리자에게 "📩 새 사용 신청" 알림 + [✅ 승인] [❌ 거절] 버튼 발송
4. 미니앱 화면에는 문서 생성 화면 대신 **"사용 승인 대기 중입니다"** 안내가 표시됩니다. 승인 전까지는 실제 기능(문서 생성)을 사용할 수 없습니다.

### 2-2. 승인 후

- 관리자가 [✅ 승인] 버튼을 누르면 신청자에게 "✅ 승인되었습니다! 메뉴 버튼으로 미니앱을 사용하실 수 있습니다." 메시지가 갑니다.
- 이후 메뉴 버튼으로 미니앱을 열면 정상적으로 문서 생성 화면이 뜹니다.
- **한 번 승인되면 재승인 없이 계속 사용 가능**합니다(만료 없음). 다만 하루 생성 횟수 제한(5회, KST 자정 리셋)은 계속 적용됩니다 — 관리자 계정만 예외.

### 2-3. 화면에 뜨는 에러 메시지 안내

| 상황 | 화면에 뜨는 메시지 |
|---|---|
| 텔레그램 밖(일반 브라우저)에서 링크를 직접 열었을 때 | "텔레그램 앱을 통해 접속해주세요. (브라우저에서 링크를 직접 열면 사용할 수 없습니다)" |
| 승인 대기 중일 때 | "사용 신청이 접수되었습니다. 관리자 승인 후 다시 시도해주세요." (또는 이미 대기 중이면 "사용 승인 대기 중입니다...") |
| 그 외 일시적 오류 | "문서종류를 불러오지 못했습니다. 잠시 후 다시 시도해주세요." |

## 3. 관리자 명령어

봇 채팅창(`@safety_rag_bot`과의 1:1 대화)에서 관리자 계정으로만 입력해야 응답합니다. 관리자가 아닌 사용자가 아래 명령어를 보내면 봇은 아무 반응도 하지 않습니다(명령어 존재 자체를 노출하지 않기 위함).

| 명령어 | 기능 |
|---|---|
| `/stats` | 승인된 사용자 수, 사용자별 생성 건수·문서유형·API 비용(추정) 조회 |
| `/authlog` | 최근 인증 실패(401/403) 내역 조회 — 누가, 언제, 왜 거부됐는지 |
| `/revoke <user_id>` | 해당 사용자의 승인을 취소. `user_id`는 `/stats` 결과에서 확인. 취소 시 대상자에게도 알림 발송됨 |

**일반 사용자 명령어**

| 명령어 | 기능 |
|---|---|
| `/start` | 승인 신청(수동). 미니앱을 여는 것만으로 자동 신청되므로 필수는 아니지만, 봇 채팅창에서 직접 입력해도 동일하게 동작 |

**명령어는 아니지만 함께 동작하는 것**

- **승인/거절 인라인 버튼**: 신규 신청 알림에 붙는 버튼. 관리자만 클릭 가능.
- **메뉴 버튼("서류 생성")**: 미니앱을 여는 버튼.

---

## 4. 승인 취소가 필요할 때 (`/revoke`)

```
/revoke 123456789
```

- 성공: "✅ 승인 취소 완료" + 대상자에게 "이용 승인이 취소되었습니다. 다시 이용하시려면 /start로 재신청해주세요." 발송
- 대상 id가 승인 목록에 없으면: "승인 목록에 없는 id입니다"
- 다시 사용하려면 대상자가 미니앱을 다시 열거나 `/start`를 보내 재신청 → 관리자가 재승인해야 함
