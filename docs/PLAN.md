### 검색 품질 검증 (Voyage-3 기준)

세 가지 질의로 교차 검색 테스트 수행, 결과를 Phase 3 비교실험용 테스트 케이스로 확정:

| 질의 | 기대 결과 | 실제 결과 |
|---|---|---|
| 정보통신공사 현장에서 안전 관련해서 뭐부터 준비해야 해? | 5개 문서 고르게 분포 (광범위 질의) | ✅ 일치 |
| 굴착 작업 관련 서류 뭐 필요해? | 작업계획서/위험성평가/관리비 상위권 | ⚠️ 관리비 문서가 top-5에서 누락됨 |
| 교육 계획은 어떻게 짜야 해? | 교육가이드가 top-1, 2 | ✅ 일치 |

→ **관리비 문서 누락**은 Phase 3에서 다른 임베딩 모델로 재검증할 핵심 관찰 지점으로 기록.

## Phase 3 — 계획 (착수 전)

### 목표
동일한 테스트 케이스 세트로 임베딩 모델 간 검색 품질을 비교하고, Phase 2에서 발견한 "관리비
문서 누락" 문제가 모델에 따라 개선되는지 확인한다.

### 비교 대상 후보
1. **Cohere embed-multilingual-v3** — API 키 발급만으로 비교 가능, 설정 난이도 낮음
2. **한국어 특화 오픈소스 KURE-v1** — 무료, 로컬 호스팅 필요, 설정 난이도 높음

### 권장 착수 순서
1. Cohere부터 시작 — 여러 임베딩 모델을 갈아 끼울 수 있도록 `common.py` 구조를 모델 분기
   가능하게 리팩터링 (`embed_with_voyage()`, `embed_with_cohere()` 분리)
2. 결과물은 `embeddings.json`을 덮어쓰지 않고 `embeddings_cohere.json` 등으로 모델별 분리 저장
3. 동일 테스트 케이스 3종 + 신규 케이스로 top-5 비교표 작성
4. 여유가 되면 KURE-v1까지 확장
5. (선택, 더 나중) Contextual Retrieval, Reranking 실험

### Phase 3 체크리스트 (착수 시 사용)

- [ ] Cohere API 키 발급 및 `.env`에 추가
- [ ] `requirements.txt`에 `cohere` 패키지 추가
- [ ] `common.py` 임베딩 함수 모델별 분기 구조로 리팩터링
- [ ] `build_knowledge_base.py`, `test_search.py`에서 모델 선택 가능하도록 수정
- [ ] 기존 3개 테스트 케이스 + 신규 케이스로 Voyage vs Cohere 비교표 작성
- [ ] "관리비 문서 누락" 케이스가 Cohere에서 개선되는지 확인
- [ ] (여유 시) KURE-v1 로컬 호스팅 환경 구축 및 동일 비교
