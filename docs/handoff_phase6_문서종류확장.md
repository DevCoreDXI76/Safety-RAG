# Safety-RAG 핸드오프: Phase 6 — 문서 종류 확장 (작업계획서 + 교육일지)

## 배경
Phase 4(텔레그램 미니앱)·Phase 5(유저 승인 + Volume 영구저장)까지 완료된 상태.
현재 `/document-types`에 위험성평가표, TBM일지 외에 안전보건교육일지·
산업안전보건관리비 사용명세서·표준작업계획서가 메뉴로는 존재하지만
생성 로직이 미완성 또는 법적 근거가 부족한 상태.

Claude.ai 세션에서 법령 원문(국가법령정보센터)을 직접 대조해서 문서별
우선순위와 지식베이스 보완자료를 정리했음. 이 문서는 그 결과를 코드에
반영하기 위한 작업 지시서.

## 이번 단계 목표
아래 2개 문서 종류의 생성 로직을 위험성평가표와 동일한 수준으로 완성한다.
**산업안전보건관리비 사용명세서는 이번 단계에서 제외** (사유는 하단 참고).

1. 표준 작업계획서
2. 안전보건교육일지

---

## 1. 표준 작업계획서

### 서브타입 (작업유형) 4종을 우선 구현
- 굴착작업
- 차량계 건설기계를 사용하는 작업
- 전기작업 (안전보건규칙 §318, 50V 초과 또는 250VA 초과)
- 중량물의 취급 작업

(5번째 확장 후보: 차량계 하역운반기계등을 사용하는 작업 — 지금은 스킵)

### 지식베이스 변경사항
- **신규 파일 추가 필요**: `표준작업계획서_법정별표.txt` (별도 첨부, 위 4개
  작업유형별 "사전조사 내용" + "작업계획서 필수기재사항" 법정 전문 포함)
- **기존 파일 정정 필요**: `표준작업계획서_가이드.txt`에서 "작업계획서
  작성 대상 작업 (예시)" 항목 중 **"밀폐공간 내 작업"을 삭제**할 것.
  법적 근거: 밀폐공간 작업은 산업안전보건기준에 관한 규칙 §38(작업계획서)
  이 아니라 §618~645(밀폐공간 작업 프로그램)의 별도 규율 대상. 이 오류를
  그대로 두면 밀폐공간 작업에 대해 잘못된 법적 근거로 문서가 생성될 위험이
  있음.
- `표준작업계획서_법정별표.txt`는 **직접 주입 방식**으로 처리
  (`위험성평가_실시규정.txt`와 동일한 패턴 — RAG 유사도 검색 대상 아님).
  선택된 작업유형에 해당하는 섹션만 프롬프트에 전체 삽입.

### UI/백엔드 변경사항
- `index.html`의 문서종류 선택 카드(`type-grid`)에서 "표준 작업계획서" 선택 시,
  TBM 선택 시 `risk-section`이 나타나는 것과 같은 패턴으로
  **작업유형 선택 UI(4개 라디오/카드)** 를 노출해야 함
- 선택된 작업유형 값을 `/generate` API 페이로드에 포함 (예: `work_type` 필드)
- 백엔드 프롬프트 조립 로직에서 `work_type`에 매칭되는
  `표준작업계획서_법정별표.txt` 섹션만 추출해 직접 주입
- 위험성평가표 연동: TBM과 마찬가지로, 작업계획서도 해당 현장의 최근
  위험성평가 회차를 선택/연동할 수 있게 하는 것을 권장 (현재 `risk-section`
  UI/API 패턴 재사용 가능)

### 테스트 우선순위
굴착작업 → 중량물취급작업 → 차량계건설기계 → 전기작업 순으로 진행
(정보통신공사 현장에서 실제 사용 빈도 기준)

## 2. 안전보건교육일지

### 서브타입 (교육종류) 우선순위
1. 신규채용시교육 (실무 빈도 최상위)
2. 특별교육 — 아래 5개 작업유형 중 선택
   - 전기작업 (75V 이상, 주의: 작업계획서의 50V/250VA 기준과 다른 수치)
   - 굴착작업 (2m 이상)
   - 맨홀작업 (§별표5 제34호 — 밀폐공간과 법적으로 별도 항목)
   - 밀폐공간작업 (§별표5 제35호 — 맨홀 외 국사/통신구 등)
   - 5m 이상 철탑 등 조립·해체·변경작업 (§별표5 제27호)
3. (후순위, 미구현 가능) 정기교육 / 작업내용변경시교육

### 지식베이스 변경사항
- **신규 파일 추가 필요**: `안전보건교육_법정시간및내용.txt` (별도 첨부,
  근로자/관리감독자 교육시간 전체표 + 특별교육 5개 항목별 교육내용 +
  건설업 기초안전보건교육 포함)
- 이 파일도 **직접 주입 방식**. 교육종류(+특별교육 시 작업유형)에 매칭되는
  섹션만 프롬프트에 전체 삽입
- 놓치기 쉬운 항목: **건설업 기초안전·보건교육(4시간)** — 건설 일용근로자
  신규 채용 시 "채용시교육"과 별도로 필요한 의무교육. UI에 별도 안내 문구
  또는 체크리스트로 노출할지 검토 필요

### UI/백엔드 변경사항
- 문서종류 "안전보건교육일지" 선택 시 교육종류 선택 UI 노출
  (신규채용시교육 / 특별교육 라디오)
- "특별교육" 선택 시 5개 작업유형 하위 선택 UI 추가 노출 (2단계 선택 구조)
- 페이로드에 `education_type`, (특별교육인 경우) `special_work_type` 필드 추가

### 테스트 우선순위
신규채용시교육 → 특별교육(맨홀작업) → 특별교육(굴착작업) → 나머지 순

## 이번 단계에서 제외: 산업안전보건관리비 사용명세서

이 문서는 다른 문서들과 근본적으로 성격이 다름 — 실제 지출 항목·금액·
증빙(영수증)이 있어야 의미가 생기는 **회계 장부형 문서**라, "작업 설명 →
AI가 위험요인을 서술" 하는 기존 RAG 생성 모델이 맞지 않음.

향후 별도 스펙으로 재설계 필요:
- AI 초안 생성기가 아니라 **입력값 검증(컴플라이언스 체크) + 서식
  포맷터**로 방향 전환
- 사용자가 실제 지출 항목(항목/금액/일자)을 입력하면, 기존
  `산업안전보건관리비_가이드.txt`의 "사용 가능/제한 항목" 기준에 맞는지
  체크하고 법정 서식으로 정리해주는 기능
- 이번 Phase 6에는 포함하지 않음. 백로그로만 기록.

## 참고: 이번에 확인한 법적 사실관계 (코드 반영 시 참고)

- 산업안전보건법 시행규칙 별표4(교육시간)는 2025.5.30 개정판이 최신
  (2026.6.1 시행 추가 개정 있었음 — 실제 반영 전 국가법령정보센터에서
  재확인 권장, 안전보건교육 관련 조문은 개정 빈도가 높음)
- 작업계획서 대상 전기작업(50V/250VA 초과, §318)과 특별교육 대상
  전기작업(75V 이상, 별표5 §17)은 **서로 다른 기준**. 두 문서 종류를
  같은 "전기작업" 라벨로 묶지 말고 각자의 임계값을 명시할 것
- 맨홀작업(별표5 §34)과 밀폐공간작업(별표5 §35)은 내용은 유사하지만
  법령상 별도 항목. 특별교육일지 생성 시 작업 성격에 따라 정확히 구분
