## 1. 규제·시장 배경 — 신뢰도 높음, 그대로 활용

중대재해처벌법이 2024년 1월 27일부로 상시근로자 5~49인 사업장까지 확대 적용된 것은 맞는 사실이고, 정보통신공사 협력업체 다수가 정확히 이 규모대에 몰려 있다는 진단도 타당합니다. "하루 평균 2시간 이상 수기 작성"이라는 페인포인트 수치는 출처가 명시되어 있진 않지만, 정성적으로는 실제 경험과 맞아떨어질 가능성이 큽니다 — **이 숫자 자체를 베타 인터뷰 첫 질문으로 던져서 직접 검증**하시는 걸 추천합니다 (역할별 체크리스트의 "기획자" 섹션 인터뷰 질문지에 추가하세요).

## 2. 경쟁사 벤치마킹 — 포지셔닝 전략에 동의

안전지키미 / 세이프로 / 안전달인 / 페이퍼리 네 곳 모두 "현장 통제 기능(출퇴근, QR서명, IoT)"까지 포함한 무거운 SaaS로 포지셔닝되어 있고, Safety-RAG가 이들과 정면승부하지 않고 "문서 생성 전용 고성능 도구"로 좁게 파고드는 전략은 지난 액션플랜에서 강조한 **유틸 서비스 원칙과 정확히 일치**합니다.

흥미로운 지점: 4곳 중 **페이퍼리(Paiperly)**가 카카오톡 연동 + 종량제/구독 혼합이라는 점에서 지금 설계 중인 BM과 가장 유사한 포지션입니다. 차별화 지점을 "정보통신공사 4대 고위험 작업(고소·밀폐공간·활선근접·굴착) 특화 + KOSHA 규정 정합성"으로 명확히 좁혀야 페이퍼리 같은 범용 툴과 헷갈리지 않습니다.

## 3. 기술 아키텍처 — 반드시 손볼 부분

### 3-1. LLM 비용 — 이전 검토와 동일하게 모델 세대만 교체

리서치 문서의 "Claude 3.5/4.6 Sonnet", "Claude 3.5/4.5 Haiku"라는 표기는 여러 세대를 뭉뚱그린 표현입니다. 지난번 확정한 대로 **Sonnet 5**(`claude-sonnet-5`, 8월까지 $2/$10, 이후 $3/$15)와 **Haiku 4.5**(`claude-haiku-4-5-20251001`, $1/$5)로 통일해서 적용하시면 됩니다. 프롬프트 캐싱으로 캐시 적중 시 입력비 90% 절감된다는 설명도 정확합니다.

### 3-2. "건당 원가 10원" — 낙관적 가정, 검증 필요 (중요)

문서유형별로 실제 출력 길이가 다르다는 점을 감안하면, 이 수치는 **Haiku로 처리하는 짧은 문서(TBM일지 등)에는 맞지만, Sonnet으로 처리하는 위험성평가표·작업계획서에는 낙관적**입니다. 대략 계산해보면:

- **TBM일지 (Haiku 4.5, 출력 ~800토큰)**: 캐시 적중 규정 텍스트 + 짧은 출력 → 약 7~8원
- **위험성평가표 (Sonnet 5, 출력 ~4,000토큰 — 다수의 위험요인 행이 들어가는 표)**: 캐싱을 적용해도 출력 토큰 비중이 커서 약 50~60원 수준

즉 문서유형 전체 평균은 "10원 내외"보다는 **문서유형에 따라 10원~60원대로 갈릴 가능성**이 높습니다. 다행히 이 수치들도 990원~8,900원짜리 크레딧 요금제 대비로는 여전히 마진율이 매우 높아 사업성 자체엔 문제가 없습니다. 다만 이 숫자를 근거로 "원가 10원이니 크레딧 990원은 99배 마진"이라는 식으로 사업계획을 세우면 실제보다 낙관적인 그림이 될 수 있으니, **개발자 역할 체크리스트에 이미 있는 "토큰 사용량 로깅"을 통해 문서유형별 실측 원가를 최우선으로 확보**하세요. 추정치가 아니라 실측치로 요금제를 확정하는 게 맞습니다.

### 3-3. 한글(.hwp) 자동화 — 아키텍처 충돌, 반드시 대안으로 교체 (가장 중요한 발견)

리서치에서 제안한 **pyhwpx는 Windows 환경 + 한/글(한컴오피스) 프로그램이 실제로 설치되어 있어야만 작동하는 라이브러리**입니다(내부적으로 `win32com`/COM 자동화 방식). 지금 배포 중인 Railway는 Linux 컨테이너이고 한/글 프로그램이 설치되어 있지 않으므로, **이 방식은 현재 인프라에서 그대로는 절대 동작하지 않습니다.** 이대로 백로그에 넣고 개발자 역할이 붙잡으면, 구현 단계에서야 이 사실을 발견하고 시간을 낭비하게 됩니다.

**대안**: `python-hwpx`라는 라이브러리가 있습니다. 순수 파이썬으로 작성되어 있고 한컴오피스 설치가 전혀 필요 없으며, `.hwpx`(개방형 XML 기반 포맷 — docx처럼 zip+XML 구조)를 직접 열고, 표를 채우고, 플레이스홀더를 치환할 수 있습니다. 여기에 더해 **`.hwpx`는 실제로 한국 행정기관의 표준 개방형 문서 포맷으로 채택된 형식**이라, 오히려 레거시 `.hwp`보다 "정공법"에 가깝습니다.

- Railway의 기존 FastAPI(Python) 백엔드에 그대로 추가 가능 — 별도 Windows 서버 불필요
- 다만 발주처·원청사에 따라 구버전 `.hwp`만 받는 경우가 실제로 있을 수 있으니, **베타 인터뷰에서 "귀사는 `.hwpx`도 받으시나요, 구버전 `.hwp`만 받으시나요"를 반드시 확인**하세요. 만약 정말 구버전 `.hwp`가 필수라면, 그때는 Windows 미니PC/VM 한 대를 별도로 띄워 pyhwpx를 돌리는 방식(비용·복잡도 증가)을 검토해야 하지만, 이건 **수요가 확인된 뒤에 투자할 일**입니다 — 지금 먼저 만들 필요는 없습니다.

### 3-4. 엑셀(.xlsx) 자동화 — 더 가벼운 대안이 있음

리서치가 제안한 SpreadJS는 Node.js 기반의 상용(유료) 라이브러리입니다. 이미 파이썬/FastAPI로 스택이 잡혀 있는 상황에서 Node.js 서비스를 새로 추가하고 라이선스 비용까지 낼 이유가 없습니다. **`openpyxl`(파이썬, 무료, 셀 좌표 바인딩 완벽 지원)**로 "위험성평가표의 [작업단계-유해위험요인-안전조치-위험도] 표를 정해진 템플릿 셀에 채워 넣는" 요구사항을 동일하게 구현할 수 있습니다. 스택을 하나로 유지하는 게 1인 개발자에게는 훨씬 유리합니다.

**정리하면 문서 내보내기 우선순위는 `PDF`(범용 미리보기/제출용) + `XLSX`(openpyxl) + `HWPX`(python-hwpx) 조합으로, 기존 백로그의 "PDF/DOCX 내보내기" 항목을 이 3종으로 구체화하시면 됩니다.** DOCX는 실제 업계 요구가 HWP/XLSX 쪽으로 확인되면 우선순위를 낮춰도 됩니다.

## 4. 비즈니스 모델 — 채택 추천

지난 액션플랜에서 "크레딧팩 vs 구독, 베타로 직접 물어보고 정하라"고 말씀드렸는데, 이번 리서치의 3원화 구조(무료체험 3건 → 크레딧 종량제 → 활성현장수 기반 구독)가 그 답으로 쓰기 좋은 구체안입니다. 그대로 베타 가설로 채택하는 걸 추천합니다.

한 가지 확인할 점: 990원짜리 초소액 결제는 PG사 건당 고정수수료 비중이 상대적으로 커서 실제 마진이 표면 마진율보다 낮을 수 있습니다. 결제 연동 시(운영/법무 역할 체크리스트) PG사의 건당 최소수수료 구조를 확인하고, 필요하면 "라이트 크레딧 990원"을 없애고 "실무 크레딧 10 (8,900원)"부터 최소 단위로 설계하는 것도 고려하세요.

## 5. GTM 전략 — 가장 강력한 부분, 최우선 검증 대상

> **2026-07-15 업데이트**: 아래 원안에서 "포항 포스코 협력사 네트워크"를 통한 접근은 **폐기**합니다. 이유는 두 가지입니다. ① 위존 '사파이어(SAPAIR)'가 이미 대기업 생산공장(울산 등)을 대상으로 실제 계약을 진행 중인 것으로 확인됐고, 이런 원청이 자체 EHS 플랫폼을 도입하면 소속 협력사가 개별 도구를 별도로 쓸 유인이 줄어듭니다. ② 개발자 본인이 재직 중인 포스코DX가 사내 안전관리 플랫폼을 자체 구축 중이라는 정보가 확인됐는데, 이는 경쟁 문제를 넘어 **고용관계상 이해상충 리스크**라 아예 접근하지 않는 것이 맞습니다. 아래 KICA 채널은 "대기업 계열이 아닌 독립 업체" 소속 기술지도위원·소장으로 한정해서 읽어주세요.

이 리서치에서 가장 값진 부분은 콘텐츠 마케팅(디스콰이엇, 블라인드 등 콜드 포스팅)이 아니라 **KICA 안전기술원 기술지도위원을 통한 B2B2C 바이럴 구조**입니다. 이 위원들은 이미 매달 여러 현장을 돌며 "위험성평가표·TBM일지 부실"을 지적하는 사람들이고, 그 지적을 받은 소장에게 "이 도구를 쓰면 지적사항을 쉽게 해결할 수 있다"고 자연스럽게 추천할 수 있는 위치에 있습니다. 이건 콜드 아웃바운드보다 전환율이 훨씬 높을 수밖에 없는 **웜 채널**입니다.

**단, 이 전략의 실행 가능성은 전적으로 "본인이 실제로 이 네트워크에 접근할 수 있는가"에 달려 있습니다.** 20년간 쌓으신 정보통신공사 업계 인맥을 감안하면, KICA 영남 지사 쪽에 실제로 닿는 인맥이 있으실 가능성이 있어 보입니다 — 다만 이건 제가 확언할 수 없는 부분이니, **직접 자가진단해보시길 권합니다**: KICA 안전기술원 영남 지사(부산·울산·경남도회, 대구·경북도회) 소속 기술지도위원 중 개인적으로 연락 가능한 분이 실제로 몇 명이나 되는지, 또는 그런 분을 소개받을 수 있는 1~2단계 인맥이 있는지부터 확인해보세요. 이때 대상은 **대기업 계열사 소속이 아닌, 독립 중소 시공업체를 주로 지도하는 위원**으로 한정하세요. 만약 이 채널이 실제로 열려 있다면, 이건 지난 마케팅 계획의 1순위 채널을 통째로 교체할 만한 발견입니다.

지자체 중대재해 예방 지원사업(경북경제진흥원 등) 연계는 리서치에 실제 공고 링크가 첨부되어 있어 존재 자체는 확인되지만, 신청 자격·시기·연 1회 여부 등은 공고 원문을 직접 확인하셔야 합니다.

## 6. 통합 액션 플랜 — 기존 로드맵 대비 변경 사항

| 기존 계획 | 이번 리서치 반영 후 |
|---|---|
| PDF/DOCX 내보내기 | → **PDF + XLSX(openpyxl) + HWPX(python-hwpx)** 3종으로 구체화. DOCX는 후순위로 밀림 |
| 디스콰이엇/블라인드 콜드 포스팅이 1차 마케팅 채널 | → **KICA 기술지도위원 웜 채널 접근 가능성부터 먼저 확인.** 접근 가능하면 이게 1순위, 콜드 포스팅은 2순위로 재배치 |
| 베타 참가자 10~15명, 채널 불문 | → **영남권(경북/경남/울산/포항)의 대기업 비계열 독립 업체를 우선 리스트업** (포스코 계열·사파이어 기계약 원청 소속은 제외 — 0-2절 참고) |
| 크레딧팩 vs 구독, 베타로 결정 | → 리서치의 3원화 구조(무료체험 3건 / 크레딧 990~23,700원 / 구독 49,000~149,000원)를 베타 가설로 채택 |
| API 비용 하이브리드 모델 | → 모델 ID만 Sonnet 5 / Haiku 4.5로 최신화, 나머지 구조 동일 |
| — (신규) | **문서유형별 실측 원가 로깅을 요금제 확정의 전제 조건으로 격상** ("10원" 추정치를 그대로 쓰지 않기) |
| — (신규) | **경쟁 환경 재평가**: 위존 사파이어(원청 대상 EHS 플랫폼) + 포스코DX 사내 플랫폼 확인 → 타깃을 대기업 비계열 독립 시공사·프리랜서 PM으로 좁힘 (메인 액션플랜 0-2절 참고) |


## 7. 지금 당장 할 3가지 (다음 스텝)

큰 계획을 다시 짜기 전에, 아래 세 가지부터 값싸게 검증하시길 추천합니다.

1. **네트워크 자가진단 (오늘, 30분)**: KICA 영남 지사 기술지도위원 또는 포항/울산 협력사 소장 중 실제로 연락 가능한 사람이 몇 명인지 적어보기 — **단, 포스코 계열사·사파이어 기계약 원청 소속은 제외.** 이 숫자가 이번 리서치 전체 전략의 실현 가능성을 결정합니다.
2. **python-hwpx PoC (개발, 반나절)**: 실제 표준작업계획서 템플릿 1개를 `.hwpx`로 만들어 python-hwpx로 필드 채우기가 되는지 먼저 검증 — 이게 안 되면 3-3절의 대안 자체가 무너지므로 가장 먼저 리스크를 제거해야 할 항목입니다.
3. **기술지도위원 1명 컨택 (이번 주)**: 실제로 반응이 오는지 가장 싸게 확인하는 방법은 큰 캠페인을 짜기 전에 딱 한 명에게 먼저 보여주는 것입니다. "지적하신 서류, 이 도구로 5분 만에 만들어드릴 수 있는데 한번 보시겠어요?" 정도의 가벼운 접근으로 충분합니다.
